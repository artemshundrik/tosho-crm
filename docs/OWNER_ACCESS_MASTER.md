## 1. Що це за система (30 секунд)

Tosho CRM — веб-застосунок: **фронтенд** (React) + **база даних/логін/файли** (Supabase) +
**хостинг і серверні функції** (Netlify), домен **tosho.pro**. Код — на GitHub. Плюс кілька
зовнішніх сервісів (AI, Telegram-боти, Нова Пошта, Вчасно). Усе нижче — щоб жоден із цих
шматків не «зник» безповоротно.

**Критичність — що станеться, якщо втратити доступ:**
- 🔴 **Supabase** — втрата = втрата ВСІХ даних (клієнти, квоти, фінанси). Найважливіше.
- 🔴 **Домен + Netlify** — втрата = сайт не відкривається / хтось перехопить адресу.
- 🔴 **GitHub** — втрата = втрата вихідного коду.
- 🟠 Решта (AI, Telegram, НП, Вчасно, Web Push) — окремі функції відваляться, але дані цілі.

## 3. Що обов'язково має бути в 1Password (чеклист)

Створи vault/розділ **«Tosho CRM»**, у ньому — окремий запис на кожен сервіс вище. Мінімум:

- ⬜ Домен (реєстратор) — логін, пароль, 2FA
- ⬜ GitHub — логін, пароль, **2FA recovery codes**
- ⬜ Netlify — логін, пароль, 2FA
- ⬜ Supabase — логін, пароль, 2FA + **Postgres DB password** + service_role key
- ⬜ OpenAI — логін, API-ключ
- ⬜ Telegram-боти — токени + який Telegram-акаунт ними володіє
- ⬜ Nova Poshta — кабінет + API-ключ
- ⬜ Вчасно — 3 кабінети (логіни + токени)
- ⬜ Web Push VAPID — пара ключів
- ⬜ Dropbox — акаунт + токени
- ⬜ **Recovery Notes** — окремий запис: «як відновити систему з нуля» (див. розділ 4)

**Хто має мати доступ до цього vault:** ⬜ ЗАПОВНИТИ (мінімум: власник + 1 довірена особа, щоб
не залежати від однієї людини).

## 4. Якщо щось сталося — план відновлення

**Сценарій A — «розробник/підрядник пішов»:**
1. Зміни паролі до Netlify, Supabase, GitHub, OpenAI (усі з розділу 2).
2. Перевір список members у Supabase/Netlify/GitHub — прибери зайвих.
3. Проверни (rotate) `service_role` ключ Supabase і `CRON_SHARED_SECRET`.

**Сценарій B — «зіпсували/видалили дані в БД»:**
1. Не панікуй — є бекапи (розділ 2.12, `docs/BACKUP.md`).
2. Відкати з останнього бекапу за інструкцією в `docs/BACKUP.md`.

**Сценарій C — «сайт не відкривається»:**
1. Перевір домен (оплата/DNS, розділ 2.1) і статус Netlify (розділ 2.3).

**Сценарій D — «втратив ноутбук / доступ до 1Password»:**
1. Саме тому в 1Password мають бути **2FA recovery codes** усіх сервісів (розділ 3).
2. І доступ до vault має мати друга довірена особа.

> Детальні технічні кроки відновлення — у `docs/DIRECTOR_ACCESS_HANDOFF.md` §"Recovery Notes"
> і `docs/BACKUP.md`.

## 5. Раз на місяць перевіряти (5 хвилин)

- ⬜ Свіжий бекап БД існує і його реально можна відновити.
- ⬜ Домен оплачений і не близько до закінчення.
- ⬜ Ліміт витрат OpenAI не пробитий.
- ⬜ У vault 1Password нічого не застаріло (ключі не мінялись без оновлення).
- ⬜ Список людей з доступом (Supabase/Netlify/GitHub) актуальний.
